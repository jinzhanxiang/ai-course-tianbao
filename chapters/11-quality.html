⚖️ L2：多 Agent 辩论 · 防幻觉最有效

🚨 关键洞察（Towards AI）："Ask an LLM to verify its own output, and it will often agree with itself — not because it has checked carefully, but because it is structurally predisposed to agree with itself."

🎭 辩论流程

Round 1：正方 Proposer → 初稿答案
           ↓
Round 2：反方 Debater → 列出 Proposer 论据漏洞
           ↓
Round 3：正方再 Defense → 反驳反方质疑
           ↓
Round 4：裁判 Judge → 第三方独立判断，投票决定

❌ 单 Agent 自检

"我检查了，没问题"——结构性偏向

✅ 多 Agent 辩论

反方 = 独立权重 + 反对立场 → 暴露漏洞

📊 MDPI 2025 数据：辩论 + 投票 → 幻觉率降低 40-60%。

🎭 OpenClaw 正反方辩论 · 实现方式

主公实测：通过 sessions_send() + 不同 system prompt 模拟"正方/反方"角色。

// 步骤 1：派 research 出初稿
sessions_send(research, "你支持 X 假设，给出 3 个论据")

// 步骤 2：派同一个 research 出反驳
sessions_send(research, "现在你反对刚才的论点，列 3 个漏洞")

// 步骤 3：派 main 做裁判
main 比对两份输出 → 不一致 → 标注"需复核"
一致 → 置信度提升

💡 实测优势

⚡ 速度
同一 session 切换立场，无需重启
🔍 覆盖
一致→确信 / 不一致→追加 review
💰 成本
仅 2× 单次推理，远低于双模型

🔍 L3：交叉验对 · Self-Consistency + Critic

🎲 Self-Consistency

同一问题问 5 次 → 投票选多数答案
原理：正确答案在不同推理路径中更稳定
局限：2 样本即可获得 90% 收益

arXiv 2025 + 2026 实证

💀 💀 幻觉的真实代价 · 主公踩过的坑

❌ 案例 1：飞马国际股东名

早期 research V1 给出"启源纳川 17.73%" → 实际是 13.73%
原因：模型对"频繁出现的旧数据"产生"自信"记忆
修正：强制 cite Wikipedia + 公告原文

⚠️ 案例 2：research 任务"口头完成"

research 二次"完成汇报"提及 AG-UI / 93 Agent / Harness 5 组件 → 文件实际未写入
原因：研究材料停留在 context，没 flush 到 disk
修正：main 强制 fs 校验 → research 改"先 write 再 announce"

⚠️ 案例 3：财务数字"看起来对"

Qwen3.6-27B 给出的"三维通信营收"与官方公告 124.89 亿元差 0.5 亿
原因：模型对小数点不敏感（幻觉的常见表现）
修正：所有财务数字必须 cite PDF 页码

💡 案例 4：SelfCheckGPT 验证

同一问题问模型 5 次，5 次答案差异 > 30% → 高度疑似幻觉
原理：模型真知道的事实，5 次回答高度一致；不知道的事，5 次会相互矛盾
应用：研究 agent 输出前自检 5 次

📌 核心数据：单 Agent 自检准确率 60-70%，多 Agent 交叉验对准确率 85-90%。差距不是统计问题，是结构问题。
