💡 什么是 Skill？不是 Prompt，是 SOP

刚接触的人很容易以为 Skill 就是「高级 Prompt」——写一段话告诉 AI 你是谁。差远了。
一份 Skill，本质是一份 SOP（标准作业程序）。它写清楚的不是「你要做什么」，而是「你该怎么做」：
· 什么时候触发（用户说了什么关键词就启动这个 Skill）
· 第一步查什么（先找哪个数据源，为什么）
· 用哪个工具（搜索？API？浏览器？哪个最合适）
· 哪一步不能跳（质量控制节点——漏了这一步报告就不可靠）
· 出错优先看哪一层（数据源挂了？API 超额？降级用哪个备选）
· 做完怎么交差（输出格式、验证标准、通知谁）
举个例子：hv-analysis Skill（纵横分析法）不是告诉 Agent「你是投资专家」，而是告诉它：先读本地文件→识别缺口→选数据源→每章自检 Q1-Q4→S1-S7 警醒信号→降级链。Agent 拿着这份 SOP，就像新人拿着带教手册——每一步都知道怎么走。

另一个例子：literature-review（文献综述）Skill
不会只扔一句「帮我综述一下」。它要求 Agent 先搜再读再写：先抓 DOI、作者、年份 → 再组织判断 → 碰上争议性结论查撤稿和后续验证 → 最后拿出来的不是论文列表，是每句话都能追溯到出处的综合评估。这就是 SOP 的力量——不是告诉 Agent「你是文献专家」，而是告诉它「先做什么、再做什么、什么不能跳过」。

后两步总计约 15 分钟。四条路径加起来不到 1 小时，你就有了一个能搜索、能查数据、能自动抓取信息的 AI Agent。

Skill ≠ 高级 Prompt。Skill 写的是「怎么做」，不是「做什么」。
一份好的 Skill = 触发条件（11个词）+ 工具选择（Python脚本）+ 步骤顺序（提交→轮询→结果→保存）+ 错误处理（失败不重试）+ 交付标准（原样输出，不加料）。

💬 第五步：学会与智能体高效对话（Prompt 工程）

环境装好了、模型配好了、技能装好了——最后一步，学会「问对问题」

📐 5 种核心提示词模式

1. 指令模式：「请用三段话总结以下内容...」

2. 角色模式：「你是一位有 10 年经验的新能源行业分析师...」

3. 示例模式：「参考以下格式输出：1.公司概况 2.财务分析 3.估值判断」

4. 步骤模式：「第一步：读取文件 第二步：提取关键数据 第三步：对比行业均值」

5. 约束模式：「不超过 500 字 · 使用表格对比 · 每个数据标注来源」

⚠️ 3 个常见 Prompt 错误

❌ 问题太模糊
→ 加角色 + 加约束，明确范围和深度
❌ 一次塞太多
→ 拆成多个子问题，逐个追问
❌ 不说输出格式
→ 给模板/示例/字数限制

📊 实战对比：差 Prompt vs 好 Prompt（星恒电源投资分析）

❌ 差 Prompt

「帮我分析一下星恒电源」

✗ 无角色定位
✗ 无分析维度
✗ 无输出格式
✗ 无数据来源要求

✅ 好 Prompt（角色 + 步骤 + 约束）

「你是一位新能源行业分析师。请基于星恒电源 FY24 审计报告（已上传），从以下三个维度分析其投资价值：
1. 行业地位（与宁德时代/比亚迪/中创新航对比）
2. 财务健康度（营收增速、毛利率、负债率）
3. 估值合理性（PE 对比、DCF 关键假设）
每个维度不超过 200 字，数据必须标注来源页数。」

🧬 案例 · 研究方法的范式转移：从「管 Agent」到「帮 Agent」

以下案例发生在2026年7月6日——从发现问题到完成范式转换，仅用了一天。这不是计划好的升级，而是一次「架构诊断→认知转变→方法重构」的完整闭环。

🔴 阶段一：V3 的结构性缺陷

17 phase × 70 模块 × 64K 行 Python
· Phase 1 输出被 Phase 2 忽略（架构断层）
· 数据源路由刚性预设（MX → Qdrant → KG 必须全走）
· 单向 12 步，不能回退补充检索
· 质疑节点是外部门禁（Agent 不知道为什么被打回）
核心问题：64K 行代码实现的是一个僵硬的流水线

🟡 阶段二：关键认知转折

一个朴素的问题：
为什么老分析师比年轻分析师强？

不是因为「步骤更多」或「模板更全」：
1. 知道什么时候该用什么数据源
2. 能识别数据缺口并主动补充
3. 会交叉验证而不是照单全收
4. 经验是原则而不是清单
核心认知：数据源应该服务 Agent，而非框死 Agent

🟢 阶段三：纵横分析法诞生

1 个 SKILL.md，0 行代码
· Phase 0 前置：先读本地一手文件，再搜
· 自适应闭环：每章 Q1-Q4 自检 → 补缺口 → 循环
· 数据源路由规则：告诉「用什么/禁止用什么」，不规定调用顺序
· 警醒信号 S1-S7：教 Agent「如何发现自己的偏见」
· 降级链：MX → iFind → SearXNG（失败不退出）
核心公式：原则 > 清单，自检 > 门禁

💡 4 个关键启示

1. Agent 需要「原则」而不是「清单」
清单越细，Agent 越蠢。给它路由规则+自检问题+降级链，让它自己判断。
2. 数据源为 Agent 服务，不是反过来
Agent 读本地文件→识别缺口→选择工具→循环。数据源是可选工具，不是必经之路。
3. 质控是「自检」不是「门禁」
不是「你写完了我来审」，而是「你写的时候问自己四个问题」。
4. 方法论演进速度快于工程实现
3 段文字、2 个模板 = 数周的工程排期。方法论就是产品。

📌 这个案例的核心启示：在 AI 辅助投资研究领域，最高杠杆的改进点不是工程自动化，而是给 Agent 一个好方法。从 64K 行 Python 到 1 个 SKILL.md，代码量降为 0，输出质量系统性超越基线——这就是方法论的力量。这和你的日常工作一样：工具很重要，但方法论才是天花板。
