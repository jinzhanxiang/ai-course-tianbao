📦 先搞清楚：什么是「技能」？

技能 = Agent 的「可安装能力包」。就像手机安装 App 获得新功能——装微信就能聊天，装地图就能导航。Agent 也一样：装「SearXNG 搜索」技能 → 能搜索网页，装「MX 金融数据」技能 → 能查财务指标。

技能商店（SkillHub）= Agent 的 App Store。浏览 → 搜索 → 看评价 → 一键安装。不写一行代码，就像你在手机上下载应用一样。

核心架构：每个技能就是一个独立的 Markdown 文件（SKILL.md），定义了这个技能「能做什么、怎么调用、需要什么参数」。Agent 读取 SKILL.md → 学会这项技能 → 下次遇到相关任务自动调用。简单、透明、可复用。

🔍 深入理解：一个 Skill 到底是什么？

刚才说了技能是「可安装能力包」。现在用一个真实案例，让你看清Skill ≠ 高级 Prompt

❌ 你以为的 Skill

就是一段高级提示词，像这样：

"你是一位资深投资分析师，请帮我分析一下宁德时代的投资价值，包括估值、财务和行业竞争格局。"

结果：Agent 自由发挥，可能查几个网页就"分析"了——没有专业数据、没有估值模型、结论不可复现。

✅ 实际上的 Skill

是一份完整的 SOP（标准作业程序），以 iFind 投研技能为例：

SKILL.md 中定义了：
· 11 个触发词自动匹配（"目标价""值得投资吗"等）
· 必须走异步任务中心，不允许 Agent 自己生成报告
· 提交 → 轮询状态（30s等待）→ 读取结果 → 保存文件
· 成功直接输出原文，禁止总结改写
· 失败输出 errorMessage，不重试

Skill ≠ 高级 Prompt。Skill 写的是「怎么做」，不是「做什么」。
一份好的 Skill = 触发条件（11个词）+ 工具选择（Python脚本）+ 步骤顺序（提交→轮询→结果→保存）+ 错误处理（失败不重试）+ 交付标准（原样输出，不加料）。
这是 Agent 和「高级提示词」之间最本质的区别。 没有 Skill 的 Agent 是一个聪明但散漫的实习生；有 Skill 的 Agent 是一个按 SOP 执行的专家。

💬 第五步：学会与智能体高效对话（Prompt 工程）

环境装好了、模型配好了、技能装好了——最后一步，学会「问对问题」

📐 5 种核心提示词模式

1. 指令模式：「请用三段话总结以下内容...」

2. 角色模式：「你是一位有 10 年经验的新能源行业分析师...」

3. 示例模式：「参考以下格式输出：1.公司概况 2.财务分析 3.估值判断」

4. 步骤模式：「第一步：读取文件 第二步：提取关键数据 第三步：对比行业均值」

5. 约束模式：「不超过 500 字 · 使用表格对比 · 每个数据标注来源」

⚠️ 3 个常见 Prompt 错误

❌ 问题太模糊
→ 加角色 + 加约束，明确范围和深度
❌ 一次塞太多
→ 拆成多个子问题，逐个追问
❌ 不说输出格式
→ 给模板/示例/字数限制

📊 实战对比：差 Prompt vs 好 Prompt（星恒电源投资分析）

❌ 差 Prompt

「帮我分析一下星恒电源」

✗ 无角色定位
✗ 无分析维度
✗ 无输出格式
✗ 无数据来源要求

✅ 好 Prompt（角色 + 步骤 + 约束）

「你是一位新能源行业分析师。请基于星恒电源 FY24 审计报告（已上传），从以下三个维度分析其投资价值：
1. 行业地位（与宁德时代/比亚迪/中创新航对比）
2. 财务健康度（营收增速、毛利率、负债率）
3. 估值合理性（PE 对比、DCF 关键假设）
每个维度不超过 200 字，数据必须标注来源页数。」

📋 可复制的 Prompt 模板（投资行业专用）

🔬 行业研究

1. 市场规模与增速
2. 前5名企业市场份额
3. 技术替代风险

总字数不超过 800 字。」
