🔧 故事 4：OpenClaw 帮 Hermes 修改配置

2026-05-08 · provider 大小写问题导致 Hermes 不认模型

问题现象

Hermes 配置文件中使用 provider: openai（小写），
但 Hermes 只认首字母大写的 provider: OpenAI

❌ 错误配置

provider: openai
model: deepseek-vash-flash

✅ 正确配置

provider: OpenAI
model: deepseek-vash-flash
结论： OpenClaw 帮 Hermes 排错，main 代理通过配置审查发现问题，修正后 Hermes 恢复正常。

🔄 Loop Engineering 工具栈趋同（2026）

Claude Code、GitHub Codex、Cursor 所有主流 Coding Agent 都在向 Loop Engineering 五元素收敛

💻 Claude Code（Anthropic）

① Automations：claude -p / wrapper 脚本
② Skills：~/.claude/settings.json
③ Memory：~/.claude/memory/
④ Worktrees：--settings 隔离配置
⑤ Eval：手动 --print 输出验证

🔵 GitHub Codex（OpenAI）

① Automations：CI/CD 集成触发
② Skills：system prompt 角色定义
③ Memory：对话 context 窗口（128K）
④ Worktrees：PR context 隔离
⑤ Eval：PR review comments

📌 趋同结论（Addy Osmani 2026-06）

所有主流 Coding Agent 都在收敛到同一套工程化框架：Automations（心跳）+ Skills（知识）+ Memory（记忆）+ Worktrees（隔离）+ Eval（判定）。
主公的 OpenClaw 体系是同款框架在企业级投资场景的完整实现。
