📋 任务分派 5 要素

main 每分派一个任务，必须包含以下 5 个要素，否则视为不规范分派：

要素 说明 示例
① 任务背景 为什么做这件事 "启源芯项目今晚投决，急需完整方案"
② 任务目标 要达成什么结果 "出一份完整投决材料，含行业/财务/估值"
③ 交付物 具体产出什么 "PDF 版投决材料 + Word 草稿"
④ 回传要求 完成后向谁汇报 "完成后回调 main 代理反馈结果"
⑤ 预期完成时间 何时应该完成 "今晚 22:00 前完成"
价值：5 要素 = 决策者"甩手掌柜"的底气。main 把脏活累活拆干净，Agent 照单执行，决策者只等结果。

🔗 sessions_send：跨 Agent 通信协议

为什么不用飞书 @？

飞书限制：机器人 @ 机器人无效。因此 OpenClaw 内置 sessions_send 作为跨 Agent 通信的唯一方式。

场景 1
决策者 → main：发一条指令
"分析启源芯，今晚投决" → main 解析意图 → 判断类型（研究类→research）
场景 2
main → research：分派任务
sessions_send(sessionKey, message, timeoutSeconds=300) → research 开始执行
场景 3
research → main：回调完成
research 完成后通过 sessions_send 回传结果 → main 汇总 → 反馈决策者
