从"问答"到"行动"——AI Agent 时代的来临

核心命题： 2024 年起，大模型从"对话"转向"行动"。智能体（Agent）= LLM + 工具 + 记忆 + 规划。

🤖 什么是智能体？

维度 传统 LLM（2023 前） 智能体 Agent（2024+）
输入 一段提示词 一个目标
输出 一段回答 一个执行结果
工具 无 函数调用 / 数据库 / API
记忆 无 短期 + 长期
规划 无 CoT / ToT / Plan-Execute
本质区别：传统 LLM 是"问答机器"，智能体是"数字员工"。

🏗️ 智能体的最小架构

🧠 大脑（LLM）

负责推理、决策、规划。MiniMax / Kimi / Claude / Deepseek / 小米 Mimo，多模型融合。

🔧 双手（Tools）

读 PDF、写文件、跑 Python、调数据库。Function Call 协议。

📝 记忆（Memory）

短期上下文 + 长期向量库。OpenClaw 用 Qdrant + bge-m3。

📋 计划（Planning）

CoT 思维链 + ToT 思维树 + ReAct 推理-行动循环。

主公的金句

"今天的 AI 不是替代人类，而是让每一个人都拥有一个'数字员工团队'。"

团队跃迁公式：个人能力 × 智能体数量 = 团队产出的乘法效应。

从今天开始，请记住 3 句话：

  1. 你不是一个人在战斗 — 你背后有 7+1 个数字员工
  2. 指令即代码 — 你发指令的方式，决定 AI 的产出质量
  3. 拍板才是你的事 — AI 负责"草稿+校验"，人类负责"拍板"

🧠 main · 总调度

职责定位：最高指挥 · 接收主公指令 · 拆分 · 分派 · 验收

5 大核心动作：

  1. 📥 接收指令 · 解析意图（提问 / 分派 / 命令 3 类）
  2. 🔪 拆分任务 · 5 要素（背景 / 目标 / 交付物 / 回传 / 时限）
  3. 📡 分派任务 · sessions_send 工具
  4. 📊 跟踪进度 · 心跳每 15min 巡检 · task-tracker.json
  5. 📋 验收反馈 · 闭环主公 · 失败立即告警

真实使用：主公 90% 的请求 · 平均每天 30+ 次调度
典型命令："今晚投决启源芯" → 90 分钟跑完 7 Agent 全流程

📋 project · 项目管家

职责定位：项目全生命周期管理 · 不做专业分析 · 只做协调推进

7 大核心能力：

  1. 📁 项目立项 · 17 类项目模板（并购 / 投决 / 储备 / 待立项）
  2. 📅 阶段管理 · 5 阶段（立项 → 尽调 → 投决 → 投后 → 退出）
  3. 💰 投资测算 · IRR / NPV / 回收期 · 敏感性自动跑
  4. 📂 文档协调 · 申请 · 起草 · 串联数据
  5. 👥 干系人管理 · 5 类角色识别
  6. ⚠️ 风险登记 · 7 大类风险自动标注
  7. 📊 可研报告 · 5 章框架自动生成

真实数据：主公 60% 的项目任务 · 当前在跑 5 个项目
代表项目：启源芯（投决）/ 星恒（尽调）/ 晶能（储备）/ 盈科（GP→LP） / 中资环（待立项）
