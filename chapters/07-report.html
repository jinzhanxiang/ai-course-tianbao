🔧 report 三大管线（OpenXML SDK）

管线 A

从零新建

研究员提供内容
→ report 从空白文档开始
→ 按国企公文规范排版
→ 输出 .docx

管线 B

编辑现有

导入已有 .docx
→ 识别并编辑内容
→ 修改、补充、格式化
→ 输出修订版

管线 C

模板填充

已有标准模板（投决模板）
→ 填入 research 数据
→ XSD 验证门控
→ 输出一致性文档

技术栈：OpenXML SDK (.NET) · XSD Schema 验证 · minimax-docx skill · 国企公文格式规范（标题/正文/页眉/页脚/页码）
