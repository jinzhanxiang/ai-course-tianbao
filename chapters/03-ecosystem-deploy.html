🛠️ 🛠️ 门槛 1 · 安装部署 · 主公破解法

痛点：在线产品很多（Claude/GPT/DeepSeek），但本地部署才是关键 — 数据不外泄 + 隐私可控 + 离线可用

🔑 破解思路 A：工具代理安装

Step 1：先装一个工具类智能体
→ 推荐：Claude Code（命令行式，最易上手）
→ 备选：DeepSeek 在线（国内访问稳定）

Step 2：用这个工具协助安装其他智能体
→ 一行命令：claude "帮我装 ollama + omlx"

效果：原本 2 天的安装 → 30 分钟完成

🔁 破解思路 B：智能体互装

核心思路：装完一个 → 让它装第二个 → 第二个装第三个...

真实路径（主公）：
① Claude Code 装 ollama（云端大模型）
② ollama 拉 Qwen3.6-27B 本地模型
③ 用 ollama 装 omlx（Apple Silicon 专属）
④ omlx 协助装 OpenClaw 整套智能体体系

效果：1 人 1 周 → 1 人 1 天 完成 5+2+1+1 体系

📋 主公实战 4 步安装链

# Step 1: 安装 Claude Code（工具）
npm install -g @anthropic-ai/claude-code
claude "请帮我安装 ollama"

# Step 2: 装 ollama（云端大模型管理）
curl -fsSL https://ollama.com/install.sh | sh
ollama pull qwen3.6:27b

# Step 3: 装 omlx（Apple Silicon 端侧推理）
# （详见 https://github.com/jinzhanxiang/omlx）
ollama run qwen3.6:27b "帮我装 OpenClaw"

# Step 4: 装 OpenClaw 5+2+1 体系
openclaw init --arch "5+2+1+1"
openclaw gateway start

🔧 故事 4：OpenClaw 帮 Hermes 修改配置

2026-05-08 · provider 大小写问题导致 Hermes 不认模型

问题现象

Hermes 配置文件中使用 provider: openai（小写），
但 Hermes 只认首字母大写的 provider: OpenAI

❌ 错误配置

provider: openai
model: deepseek-vash-flash

✅ 正确配置

provider: OpenAI
model: deepseek-vash-flash
结论： OpenClaw 帮 Hermes 排错，main 代理通过配置审查发现问题，修正后 Hermes 恢复正常。

🔄 Loop Engineering 工具栈趋同（2026）

Claude Code、GitHub Codex、Cursor 所有主流 Coding Agent 都在向 Loop Engineering 五元素收敛

💻 Claude Code（Anthropic）

① Automations：claude -p / wrapper 脚本
② Skills：~/.claude/settings.json
③ Memory：~/.claude/memory/
④ Worktrees：--settings 隔离配置
⑤ Eval：手动 --print 输出验证

🔵 GitHub Codex（OpenAI）

① Automations：CI/CD 集成触发
② Skills：system prompt 角色定义
③ Memory：对话 context 窗口（128K）
④ Worktrees：PR context 隔离
⑤ Eval：PR review comments

📌 趋同结论（Addy Osmani 2026-06）

所有主流 Coding Agent 都在收敛到同一套工程化框架：Automations（心跳）+ Skills（知识）+ Memory（记忆）+ Worktrees（隔离）+ Eval（判定）。
主公的 OpenClaw 体系是同款框架在企业级投资场景的完整实现。
