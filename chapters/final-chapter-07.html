🌊 趋势展望 · 为什么值得自己建 Agent

🏭 大厂通用工具

字节 Coze · 飞书 Claw · Kimi Claw · 阿里百炼 · 百度千帆 · 微软 Copilot Studio · Google ADK

能给你的：拖拽式编排 + 预置模板 + 零代码入门
它们的短板：
• 费用高：按 Token 计费，规模化后成本不可控
• 定制差：行业独有的数据管道无法嵌入
• 数据不可控：敏感数据上云，合规风险
• 浅层化：停留在"工具使用"，无法形成壁垒

🎯 你自建的纵深 Agent

先解放自己 → 再提升自己 → 最终差异化

你能构建的差异化壁垒：
• 行业知识库：历史项目、研报、尽调底稿
• 业务工作流：公司特有的审批、决策链路
• 经验规则引擎：投资逻辑、风控红线、估值偏好
• 持续进化：每次任务的记忆沉淀 + 反馈闭环
• 数据安全：核心数据 100% 本地，不经过第三方

💡 这就是今天这节课的价值

大厂给你的是一台电脑——每个人都能拿到。
你能不能用好这台电脑，取决于你在上面装了什么软件、存了什么数据、建了什么流程。

真正的护城河不是你用了什么模型——是你把自己的行业 Know-how 做成了模型能调用的资产。

🌊 Agent 未来三个方向

三者叠加 = 不可替代的竞争壁垒

🏛️ MOA 多 Agent 协作

提出：Together AI（arXiv 2406.04692）
原理：多层 Agent 层级协作——Proposer 生成方案 → Aggregator 综合提炼。仅用开源模型即超越 GPT-4 Omni。
启示：5+2+1+1 本质就是 MOA 的投资领域实现。未来多 Agent 协作是标配，不是可选项。

🎯 领域微调路径

技术路线：LoRA/QLoRA 单卡微调 → DPO 偏好对齐 → 合成数据训练。
三步走：持续预训练（领域语料）→ 指令微调（领域数据）→ 偏好对齐（精化）。
方向：本地 oMLX 微调 → 高度定制化的投资领域模型。

📚 知识库增强

已在实践：Data Agent 清洗流水线 → KG+Qdrant 双库入库。每次任务自动写入结构化知识。
效果：检索命中率 > 95%，5,786→5,886 实体持续增长。
本质：知识库 = 大模型的"长期记忆外挂"——模型能力不变，但可调用的知识资产持续增长。

Agent 的未来不是"一个更强的模型"—— 而是 MOA 协作让系统更聪明 + 领域微调让模型更专业 + 知识库增强让能力可持续积累。
三者叠加 = 不可替代的竞争壁垒。
