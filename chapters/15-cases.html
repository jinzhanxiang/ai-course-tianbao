⚙️ 案例 D · main + Claude Code + Hermes：协同运维

📌 背景：main 代理作为总调度，每小时心跳巡检 + 任务分发；Claude Code 负责架构诊断；Hermes 负责脚本迭代。三者形成 "监控 → 诊断 → 修复"三角闭环。

[main 调度] 每 15 分钟心跳：检查 Docker/PG/Qdrant + session 大小 + 任务超时
       ↓ 发现 research agent session 超过 10MB 告警阈值
[Claude Code 诊断] 分析 session 膨胀原因：历史 context 累积 → 建议压缩策略
       ↓ 输出压缩脚本和触发阈值
[Hermes 执行] 按建议更新 session-cleanup.sh，添加自动压缩逻辑
       ↓ 验证：session 10MB→5MB，下个心跳确认正常
[main 记录] 将本次修复写入 MEMORY.md教训库，防止同类问题再次发生
