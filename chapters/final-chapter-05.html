⚠️ 第 05 章 · 有什么风险和坑？

前面 4 章讲了智能体能做什么。本章主动讲可能出什么问题——不讲漂亮话，只讲真问题。每个风险都有具体对策和实测数据。

🛡️ 第一大风险：LLM 会产生事实错误（幻觉）

不是故意的，但结果一样严重。以下 4 个坑全部来自星恒电源项目的真实记录。

🔴 股东比例造假

LLM 声称某股东持股 17.73%，Tavily 交叉验证发现无此数据——数字从一个案例"迁移"到了另一个公司。

🟡 research 声称"已完成"

两次 sessions_send "已完成"，但文件实际未写入。main 通过文件校验发现差异并仲裁。

🟣 财务数字年份错配

LLM 将 2022 年营收 35.99 亿标注为 2024 年。原始数据源标记模糊导致。

🔵 SelfCheckGPT 一致性验证

10 个 LLM 输出做事实一致性检查，3/10 存在偏差。自检是第一道防线。

📋 防线：cite-then-answer · 先引用再回答

1. 来源锁定

每条关键数据标注来源文件 + 段落。例："（星恒 FY24 审计报告 P23 营收 36.36 亿）"

2. 双重验证

关键数字必须两个独立源交叉验证。一个来自 LLM 检索，一个来自 Tavily / 外部 API。

3. self-check 自检

输出后三项检查：事实核查 / 逻辑一致性 / 来源可追溯。三项通过才算"完成"。

📌 实战效果：引入 cite-then-answer 后，幻觉类错误从 ~15% 降至 <3%。关键不是"更聪明的模型"，而是更严格的流程。

⚠️ 第二大风险：以为"加功能就是变好"

真实案例：在纵横分析法（HV Analysis）基础上叠加 V3 质疑 Agent 后，报告质量不升反降。

📐 基线：HV Analysis

晶能光电 · 65,265 字 · 49 个 H2 · 11 个 Insight · 445 数据点 · 6 家对标公司

🔬 增强版：+ 质疑 Agent

叠加上去后 → 11,136 字（-83%）· 14 个 H2（-35）· 0 个 Insight（-100%）· ~50 数据点（-89%）

🔴 根本原因：工作流被"腰斩"

指令"不走 data 代理"被误解为"不进行 LLM 深度分析"→ 检索层被跳过 → 对标分析被跳过 → 只做了数据溯源标注（唯一提升项）。新模块把原有功能"吃掉"了。

🟢 修复：框架层面集成，不是流程外打补丁

新增 --input-dir 参数 + local_context 注入到 LLM prompt → 保留完整工作流。本地文件作为 context，不替代分析步骤。

📌 核心启示：AI Agent 的升级不是搭积木。新模块会改变工作流执行路径，集成方式不对，新功能会把原有功能"吃掉"。引入新工具要先改造流程，而不是把新工具硬塞进旧流程。

🧬 第三大风险：以为"代码越多越可靠"

从 64K 行 Python 的刚性流水线 → 1 个方法论文件的范式转移。一天之内完成的认知跃迁。

🔴 V3 流水线

17 phase × 70 模块 × 64K 行 Python。Phase 1 输出被 Phase 2 忽略（架构断层）。数据源路由刚性预设。单向 12 步不可回退。代码越多，越僵硬。

🟡 认知转折

为什么老分析师比年轻分析师强？不是步骤更多，而是：知道什么时候用什么数据源 / 能识别数据缺口 / 会交叉验证 / 经验是原则不是清单。

🟢 hv-analysis

1 个方法文件 · 0 行代码 · 自适应闭环（每章自检 → 补缺口 → 循环）· 内置警醒信号 · 降级链。核心公式：原则 > 清单，自检 > 门禁。

维度 V3 工作流 纵横分析法
定位给流水线的配置文件给 Agent 的方法论
数据源刚性路由：固定调用顺序自适应路由：Agent 自主选择
质控外部门禁（不知道为何被打回）内嵌自检（写的时候问自己）
代码量64K 行 Python0 行（纯方法论文件）

📌 核心启示：在 AI 辅助投资研究领域，最高杠杆的改进点不是工程自动化，而是给 Agent 一个好方法。工具很重要，但方法论才是天花板。（V3 vs hv-analysis 的架构对比详见第06章 Demo 2）

📋 5 条踩坑教训 + 成本真相

⚠️ 先做对的事，再正确地做

坑 1：一开始就做模型微调 → 浪费 3 个月 → 先做编排，验证流程
坑 2：追求"完全自主 Agent" → 失控 → 人在回路，先做 80 分再迭代
坑 3：不做标准流程直接复制团队 → 报错 → 先沉淀 SOP 再培训
坑 4：忽略记忆功能 → Agent 失忆 → 每个 Agent 配独立记忆文件
坑 5：没有 main 调度 → 多 Agent 抢活 → 必须有一个总指挥

🔥 决策者一定会问的 5 个问题

从大量实际交流中提炼。每个问题有具体数据支撑，不回避不粉饰。

Q1 AI 真能写好深度报告吗？

单 Agent 不行，5 Agent 协作就能。传统 13 份研报 3 天 → 2 小时。主报告 ~64K 字，OCR 准确率 96.8%。

Q2 模型是不是越贵越好？

不是。三段式选型：80% 简单任务 → DeepSeek+GLM，15% 主力 → MiniMax M3，5% 复杂 → Claude Sonnet / DeepSeek V3/R1。

Q3 系统稳定性怎么保证？

3 层保障：Agent 互校验 → Wiki 检索交叉验证（5,894 实体 · 11,075 关系 · 193 份研报 · 99.7% uptime）→ 决策者最终判断。人是底线，AI 是放大器。

Q4 新人怎么快速入门？

5 阶段路径：手写 → AI 辅助 → AI 主导 → Agent 协作 → Agent 自治。新人入门从 3 个月 → 2 周。

Q5 AI 输出的东西我能直接用于决策吗？

不能，也不应该。AI 输出是参考资料，不是决策依据。正确用法：AI 提供数据、模型和初步分析 → 投资团队交叉验证关键数据 → 结合行业经验和商业判断做最终决策。AI 负责效率，人负责质量。所有数据必须可追溯到原始来源。

核心认知：智能体的竞争，短期看模型能力，长期看记忆管理。谁能让 Agent 记住更多、忘得更少、检索更快，谁就拥有更强的 AI 劳动力。
