📄 投资请示起草

project 负责起草投资请示文档框架，调用 research 出行业分析、data 出财务数据、report 做最终排版。main 负责调度各环节协同。

📅 节点跟踪

每个项目关键节点（立项/尽调/投决/投后）都有时间戳记录。主公随时问 project，项目当前在哪一步，一目了然。

🔗 GP/LP 双线

盈科汇金等多个基金项目涉及 GP（普通合伙人）+ LP（有限合伙人）双线管理。project 负责维护 GP→LP 关系和出资结构。

🤝 跨 Agent 协调

project 接到 main 分派 → research 做研究框架 → data 清洗材料 → report 排版输出。全程不需要主公介入。

🧠 main · 总调度

职责定位：最高指挥 · 接收主公指令 · 拆分 · 分派 · 验收

5 大核心动作：

  1. 📥 接收指令 · 解析意图（提问 / 分派 / 命令 3 类）
  2. 🔪 拆分任务 · 5 要素（背景 / 目标 / 交付物 / 回传 / 时限）
  3. 📡 分派任务 · sessions_send 工具
  4. 📊 跟踪进度 · 心跳每 15min 巡检 · task-tracker.json
  5. 📋 验收反馈 · 闭环主公 · 失败立即告警

真实使用：主公 90% 的请求 · 平均每天 30+ 次调度
典型命令："今晚投决启源芯" → 90 分钟跑完 7 Agent 全流程
