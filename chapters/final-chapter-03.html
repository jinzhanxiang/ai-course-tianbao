身份·记忆·技能 · 一个 Agent 的完整构成

第 02 章用星恒案例展示了「能做什么」。本章用5 分钟讲清楚背后的系统是怎么运转的——不深入技术细节，重点是让你理解：数据安全吗？谁在控制？出错了怎么办？

🏗️ 一个智能体的最小组成——身份第一

不是「一个会聊天的 AI」——核心是身份。你是谁 → 你有什么能力 → 你用什么工具 → 你怎么思考

🪪

身份（Identity）

"我是谁？"——最底层、最关键的设定。身份模糊 = Agent 摇摆不定。OpenClaw 用 identity/rd.md 定义

🧠

大脑（模型）

推理、决策、规划。云端主力 + 本地兜底。模型越强，Agent 越聪明

🔧

双手（工具+技能）

读 PDF、写文件、跑 Python、搜网页。工具是原子操作，技能是打包的能力

📝

记忆（Memory）

短期会话 → 中期项目标签 → 长期知识沉淀。越用越聪明，不是比喻

为什么身份是第一位的？如果没有身份，Agent 就是一个「什么都能聊」的通用对话机器人——它可以聊天气、写诗、编故事，但不会知道自己是投资研究团队的成员，不会坚持投资研究的严谨标准。身份定义了 Agent 的全部行为边界。OpenClaw 的 identity/rd.md 文件、Claude Code 的 CLAUDE.md 文件，本质上都是回答同一个问题："你是谁？"

🌙 OpenClaw Dreaming 系统：每日"做梦"

浅睡（每次任务后）：即时摘要 → 本次对话的关键决策和产出
打盹（周期性）：合并同类上下文 → 减少冗余记忆
做梦（每日深度处理）：全天对话日志 → 提取教训 → 更新 memory/YYYY-MM-DD.md 每日笔记
结果：Agent 每天早上"醒来"，知道昨天做了什么、学到了什么。

🧩 主流记忆插件：Mem0 / MemOS

Mem0 (mem0.ai)：开源记忆层，自动从对话中提取实体/关系/偏好，跨会话持久化。每个用户/Agent 独立记忆空间。
MemOS：记忆操作系统，类比"记忆体"——统一管理短期/长期/情景/语义四类记忆。
本系统实践：MEMORY.md + Memos (SQLite) + Obsidian Vault → 不依赖外部服务，本地完全控制。

📂 长期记忆的具体实现：每日笔记 + 工作底稿 + Obsidian

每日笔记（memory/YYYY-MM-DD.md）：每次工作结束后自动生成 → 记录今天的对话摘要、关键决策、产出文件。不是人写的，是 Agent 睡觉时自动写的。
工作底稿：项目过程中产生的研报、模型、图表 → 沉淀到 Obsidian Vault，下个项目 Agent 可以按需读取。
联动机制：SessionStart Hook → 自动加载今日笔记 + 上次会话摘要 → Agent 一睁眼就知道"我在哪、在做什么、昨天做到哪了"。
→ 这些不是概念——是本系统实际运行的记忆机制。MEMORY.md (3.5KB) + Memos (#标签检索) + Obsidian (50+ memory 文件)

📦 技能（Skill）= Agent 的"App"

如果 6 个 Markdown 文件定义了 Agent 的"身份和性格"，技能就是 Agent 的"可安装能力包"

🧩 技能 vs 工具 · 有什么区别？

工具（Tool）= 原子操作
读文件、写文件、发 HTTP 请求、执行 Shell 命令——单一功能，无上下文
例如：read_file、web_search、bash
技能（Skill）= 打包的能力
工具 + Prompt + 知识 + 流程——完整的工作流，自带上下文和自检
例如：hv-analysis 技能 = SearXNG搜索 + MX财务 + 浏览器 + 纵横分析模板 + 自检清单
类比：工具是螺丝刀、扳手——原子工具。技能是「修车」——需要知道先后顺序、用量多少、注意事项。你不需要教 Agent 怎么拧螺丝（它有工具），你需要告诉它怎么修车（给它技能）。

为什么不用「1 个 AI 临时派生 4 个子代理」？
临时派生 = 无身份、无记忆、无成长——每次任务从零开始。就像每次招一批临时工。
为什么用「5 个独立 Agent」？
独立身份 + 独立记忆 + 技能沉淀。每个 Agent 越用越强，上下文隔离 + 并行执行实现数量级加速。
📌 一句话：不是因为 5 Agent 架构更「先进」——是因为专业技能需要持久身份来承载。持续积累才能形成真正的专业能力。
