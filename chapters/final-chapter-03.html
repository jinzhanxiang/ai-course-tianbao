📝 记忆系统深度解析 · Agent 如何记住一切

不是「一个记忆文件」——是三层分工、逐级降本的记忆架构。以下来自超级单体智能体的真实运行数据

⚡
短期 · 会话级

每次会话自动注入 3.5KB，仅保留当前活跃上下文

MEMORY.md (3.5KB)
自动注入——每次会话启动时自动加载
仅保留当前活跃上下文：项目状态、关键决策、待办事项
会话压缩时可丢弃（不损失长期知识）
→ Token 基准：3.5KB（极轻）
🏷️
中期 · 项目级

按标签搜索精准加载，Token 节省 70%+

Memos (SQLite + API)
按标签搜索，精准加载
决策/教训/洞察，每条带结构化标签
只加载相关标签（如 #decision #lesson #xingheng）
→ Token 节省 70%+（对比全量加载）
📚
长期 · 知识级

按需读取不预加载，Token 节省 90%+

Obsidian (Markdown)
按需读取，不预加载
研报/实体/指标等结构化知识库
通过 read_file 在需要时才加载
→ Token 节省 90%+（不预加载）

📊 同一任务的三层加载对比

层级 存储工具 触发方式 Token 消耗 vs 全量加载
即时 MEMORY.md 自动注入 ~3.5KB 基准线
中期 Memos #标签 搜索 ~1,800 tokens 节省 70%+
长期 Obsidian read_file 按需 ~300 tokens 节省 90%+

🔄 实际数据流：星恒项目中的一次记忆调用

1. SessionStart Hook → 自动注入 MEMORY.md (3.5KB)：当前项目状态 + 关键决策 + 上次会话摘要
2. UserPrompt: "做星恒尽调" → 触发关键词 → Memos 搜索 #xingheng #valuation #battery
3. Memos 返回 1,800 tokens：星恒历史决策 · PE估值教训 · 产业链分析框架
4. Agent 按需 read_file → Obsidian：星恒底稿 · 券商研报 · DCF模型（仅 300 tokens 上下文）
5. 全流程 Token 总量：~5,600 tokens vs 全量预加载需 ~20,000+ tokens → 实际节省 72%
核心设计原则：分层记忆，减少重复加载。不是把所有东西都塞进上下文——而是在需要的时候，加载需要的东西。这就像你不会把整个档案室搬进会议室——你只是调出跟当前议题相关的那几份文件。

📋 计划与行为规范 · 文件即架构 = Agent 的「宪法」

前面说到四器官中的「计划」——但计划不是凭空发生的。计划是在这 5 个文件的约束下运行的。文件定义了 Agent 的身份边界和能力范围，计划是在这个框架内的动态推理。

🧠 计划（运行时动态行为）

❌ 计划 ≠ 文件即架构
计划是 Agent 运行时的动态行为——每一步推理、每一次工具调用、每一个任务拆解
✅ 文件即架构 = 计划的「宪法框架」
5 个 Markdown 文件定义了 Agent 的全部行为边界——计划只能在这个边界内进行。就像一个人的性格+技能+经验限定了 TA 的思考方式
🔗 关系：文件约束计划，计划驱动行动
AGENTS.md 说「任务分派规则」→ 计划拆解任务时循此规则
SOUL.md 说「诚实优于讨好」→ 计划推理时不回避负面结论
TOOLS.md 说「工具调用上限 30 次」→ 计划控制在 30 步以内

为什么不用「1 个 AI 临时派生 4 个子代理」？
临时派生 = 无身份、无记忆、无成长——每次任务从零开始。就像每次招一批临时工。
为什么用「5 个独立 Agent」？
独立身份 + 独立记忆 + 技能沉淀。每个 Agent 越用越强，上下文隔离 + 并行执行实现数量级加速。
📌 一句话：不是因为 5 Agent 架构更「先进」——是因为专业技能需要持久身份来承载。持续积累才能形成真正的专业能力。

🔒 数据安全 + 系统稳定 · 自动化与生态

🔧 5 层自动化 Hooks + 8 个插件生态

🪝 5 层 Hooks（全生命周期自动化）

SessionStart 加载 → UserPromptSubmit 关键词检测 → PreCompact 落盘提醒 → PreToolUse 同步 → Stop 双向持久化

🪝 5 层 Hooks（全生命周期自动化）

SessionStart：自动加载项目索引 + 当前状态 + 经验教训 + 会话日志
UserPromptSubmit：关键词检测 → 自动加载相关上下文
PreCompact：上下文压缩前强制提醒落盘，防信息丢失
PreToolUse：任务进度实时同步到 Obsidian 知识库
Stop：会话结束时双向同步 + 追加时间戳
🔌 8 大插件（数据源 + 通道全覆盖）

SearXNG 自建搜索、研报客、飞书文档、浏览器自动化、Tavily、ACP 路由、飞书 Wiki、飞书云盘

🔌 8 大插件（数据源 + 通道全覆盖）

🔍 SearXNG 自建搜索：Docker 自托管元搜索引擎，多引擎聚合（Google/Bing/DuckDuckGo 等），LLM 前处理（查询改写+意图识别）+ LLM 后处理（相关性评分 0-10 + 噪声过滤 + 域名黑白名单 + 安全网兜底），无 API 配额限制
