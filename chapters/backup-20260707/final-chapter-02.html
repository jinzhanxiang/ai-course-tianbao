🤖 什么是智能体？

先看清楚这场变革 —— 智能体时代 vs 传统时代，到底变了什么？

变革大背景：我们正在经历 「软件 2.0」 的变革。从「手工生产」→「智能工厂」需要三步：云计算 (2010s) → 大模型 (2023) → 智能体 (2024+)。

📊 智能体时代 vs 传统时代（5 个核心差异）

维度 传统软件时代（2000-2023） 智能体 Agent 时代（2024+） 变革
交互方式 点按钮 / 填表单 / 下拉菜单 说话下达目标 从 UI 到 NL
使用主体 人被软件指挥 人指挥软件干活 主体反转
输出类型 信息展示 / 表格汇总 执行结果 / 交付物 从查到做
调用关系 单个软件独立运行 多个 Agent 并行协同 + 反向调用人 从点到网
能力交付 「点击什么出什么」 「赋予一个可用的智能能力」 从能用到能动

🌍 拿 3 件事看变革

📊 出一份投资周报

传统：打开 Wind → 导出持仓数据 → 手动截图 → 粘贴到 PPT → 写分析 → 排版 → 发邮件
智能体：一句话：「汇总本周持仓变动+行业新闻+下周关注」，Agent 自动拉数据、写分析、生成 PPT

📑 出一份尽调报告

传统：50 页 Word → 手工查财报 → 手工录数据 → 手工改格式
智能体：一句话：「出星恒电源尽调报告」→ 7 维度并行 90 分钟出 80 页成稿

🧾 清洗 13 份研报

传统：实习生请假 3 天手工 OCR + Excel
智能体：data agent 2 小时 → 79 实体 / 110 指标 / 178 逻辑 / 自动入库

💬
① ChatBot 时代
关键词：知识百科 · 单轮对话
一句话能聊，一件事做不了。不能读文件、不能跑代码、不能改数据库。
代表：ChatGPT、文心一言、通义千问
🪄
② Copilot 时代
关键词：贴身副驾 · 半自动
人在回路：AI 起草，人审核。提效 30-50%，但每一步都要人点头。
代表：GitHub Copilot、Cursor、Microsoft 365 Copilot
🤖
③ Agent 时代
关键词：目标驱动 · 全自动单兵
你定目标，它自己拆任务、调工具、交付结果。Agent = LLM + 工具 + 记忆 + 规划。
代表：Devin、AutoGPT、Manus、Computer Use
👥
④ 协同时代 ⭐
关键词：数字团队 · 联合作战
main 调度 + 多个专业 Agent（研究 / 数据 / 报告 / 项目）分工协同，完成单一 Agent 干不了的任务。
代表：多智能体协同体系 · 本节课要讲的体系
🦾
⑤ 具身智能
关键词：物理世界 · 走进现实
Agent 从屏幕走进物理世界：自动驾驶、人形机器人、四足机器狗。通用人工智能 (AGI) 的关键拼图。
代表：Tesla Optimus、Figure 02、宇树 H1、小鹏 PX2
「从聊天到干活 · 从单兵到团队 · 从屏幕到物理世界」
我们正处在 第 ④ 阶段 的起点 —— 多 Agent 协同 + 长期记忆 + 垂直行业落地
理解这个演化路径，你才能判断：一个 AI 工具是"玩具"还是"生产力"——关键看它处在这条线的哪个位置。

🧭 智能体工具全景图 · 6 大分类

市面上工具繁多，按定位分为六类——选对类型比选对产品更重要

💻 编程助手类

IDE 集成 · 代码生成 · 单兵提效

代表：Claude Code、Cursor、GitHub Copilot、通义灵码
特点：深度嵌入开发环境，实时补全、重构、调试。适合有编程基础的个人提效。
⚠️ 注意：使用境外工具时注意代码不上传敏感项目

🏗️ 多智能体框架

开源协同 · 主从调度 · 团队作战

代表：开源多智能体框架、LangGraph、CrewAI、AutoGen
特点：多个专业 Agent 分工协作，main 调度 + worker 执行。复杂项目必备，单一 Agent 干不了的活交给它。
✅ 本课程核心：5+2+1+1 多智能体协同体系

⚙️ 轻量执行类

脚本自动化 · 工作流 · 快速迭代

代表：Hermes（商汤大模型 + DeepSeek V4 Flash）
特点：轻量、免费 API、无需编程基础。适合数据清洗、日报生成、信息爬取等重复性任务。
✅ 本课程赠送：Hermes 安装包 + 3 个工作流模板

🏭 国内平台类

零代码 · 拖拽式 · 国内合规

代表：扣子（Coze）、通义千问百炼、文心智能体、Kimi 平台
特点：拖拽搭建、预置模板、零代码入门。适合非技术用户快速上手，但深度定制受限。
定位：通用场景覆盖（客服/营销/数据分析）

🖥️ Claw 系 CLI 工具

终端原生 · 文件操作 · 工程化

代表：Claude Code、Kimi Claw、飞书 Claw
特点：命令行 AI 编程助手，直接操作文件系统、执行 Shell、管理 Git。工程化能力极强，但需要命令行基础。
各大厂商纷纷推出自家 Claw——这是 AI 编程助手的标准化赛道

🔑 选型关键区别

🟢 单兵（编程助手/Claw）vs 团队（多智能体框架）
🟢 云端（平台类/Claw系）vs 本地（开源框架可本地部署）
🟢 通用（扣子/百炼）vs 垂直（本课程投资行业定制）
🟢 零代码（国内平台）vs 需编程（Claw系/开源框架）
核心判断：复杂度决定工具选择。简单任务用平台/轻量工具，复杂项目上多智能体框架。

📌 本课程的定位：编程助手（Claude Code）+ 轻量执行（Hermes）+ 多智能体框架（5+2+1+1）三合一。不是选一个，是按场景组合使用。

🏗️ 智能体的最小架构

一个智能体 = 大脑 + 双手 + 记忆 + 计划 —— 缺一不可

🧠

大脑（LLM）

负责推理、决策、规划。
MiniMax / Kimi / Claude / Deepseek / 小米 Mimo，多模型融合。

🔧

双手（Tools）

读 PDF、写文件、跑 Python、调数据库。
Function Call 协议。

📝

记忆（Memory）

短期上下文 + 长期向量库。
本体系用 Qdrant + bge-m3。

📋

计划（Planning）

CoT 思维链 + ToT 思维树 + ReAct 推理-行动循环。

这套最小架构落到投资行业，自然扩展为 5+2+1+1 体系：
投资工作流天然分工——调度→研究→数据→项目→报告，每个环节由独立 Agent 负责。 外加两个工程助手（编码 + 脚本执行）、一个知识中台（积累项目经验）和一个本地模型（数据安全兜底）。 下一章展开这个架构的完整设计。
