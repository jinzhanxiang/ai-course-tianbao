📅 12 个月升级里程碑

📅 第 1 月（零基础→用户）
  • 注册 ChatGPT / Claude，提问 50 个真实问题
  • 学 5 种提示词模式（指令/角色/示例/步骤/约束）
  • 目标：让 AI 写出 ≥80% 可用初稿

📅 第 2-3 月（用户→制作者）
  • 学 Dify / n8n，搭 3 个工作流
  • 接入 1 个外部 API（邮件/日历/微信）
  • 目标：自动化日常任务 80%

📅 第 4-6 月（制作者→训练官）
  • 沉淀 5 个 标准流程，团队培训
  • 引入记忆 + 反馈机制
  • 目标：团队整体效率 2x

📅 第 7-12 月（训练官→架构师）
  • 设计 5+ Agent 协同体系
  • 跨部门流程打通
  • 目标：人均产值 3x

⚠️ 5 条升级踩坑教训

❌ 坑 1：一开始就模型微调 → 浪费 3 个月 → 先做编排，验证流程
❌ 坑 2：追求"完全自主智能体" → 失控 → 人 + 智能体协同，留人在回路
❌ 坑 3：不做 标准流程 直接复制给团队 → 报错 → 先沉淀 标准流程 再培训
❌ 坑 4：忽略"记忆"功能 → 智能体失忆 → 每个智能体配记忆文件
❌ 坑 5：没有 main 调度，多智能体抢活 → 必须有一个 main 总指挥

📌 LinkedIn 2026："99% automation, 0% agency — that's the current reality of AI agents." 别被"自主智能体"宣传骗了。

🎁 星恒电源项目 × 4 个智能体实战拆解

一个项目，四个视角。以下工作流均可直接复用到你的项目——换一个项目名，流程不变。

🔬 research + Claude Code · 深度研究

星恒项目做了什么：

• 动力电池产业链 7 维度并行研究
• 历史基准 IRR 模型（复合增长率 0-15%）
• Monte Carlo 10000 次模拟 → P10/P50/P90
• 估值 36.4 亿，结论：亏 68%，不建议收购
📐 计算链路： 历史基准IRR模型（复合增长率 0-15% / 净利率 1.5-2.5% / PE 8-12倍）→ 全部组合深度亏损 → Monte Carlo 10000次模拟 → P10/P50/P90 全分布 → 结论：估值 36.4 亿，较收购价亏损约 68%
关键迭代：v1 基于公开信息 → 营收高估约 100%。v2 引入审计数据修正。v3 产业链上下游全景分析。
可复用模板：输入公司名 → 自动出行业+财务+估值三合一报告。传统 3 天 → 90 分钟。

📄 report · PPT + 公文排版

星恒项目做了什么：

• 80 页尽调报告 Word 自动排版
• 党委会汇报 PPT 一键生成（含图表）
• 股权穿透图自动渲染（50 家股东）
• GB/T 9704-2012 国企公文标准
关键迭代：初版图表数据与正文不一致 → 引入 report 自检机制 → 交付前自动校验数据引用准确性。
可复用模板：Markdown → DOCX + PPT + PNG 预览。3 分钟完成 2-3 天排版工作。

📋 project · 全流程管理 + 线上部署

星恒项目做了什么：

• 立项 → 节点跟踪 → 版本控制 → 交付
• 尽调底稿、审计报告、估值模型集中管理
• GitHub Pages 一键部署：报告链接直接发同事
• 项目复盘：全部过程记录可追溯
关键迭代：初版无版本控制 → 尽调底稿分散在多个文件夹 → project Agent 统一纳管 + GitHub 版本追溯。
可复用模板：新项目 → 复制模板 → 改项目名 → Agent 自动跟踪全流程。10 个项目库已就绪。

🗄️ data + Hermes · 研报获取→筛选→清洗→入库

星恒项目做了什么：

• 50 家股东信息从天眼查自动提取
• 动力电池产业链研报 OCR → 结构化入库
• 竞品（宁德时代/比亚迪/中创新航）数据对比
• +98 实体入 Wiki，下次同类项目直接检索
📡 阶段1 · 研报客API搜索+五维加权初筛：15-20组关键词搜索 → 解析文本输出 → UUID去重 → 加权评分（页数25/年份20/机构权威15/深度指标8/主题相关8）→ 高置信度机构白名单（工信部/能源局/GGII/头豹/赛迪/BCG等20+机构）
🔍 阶段2 · 本地二级筛选：文件≥2MB+页数≥20 → 排除个股报告/周期点评/宏观非主题/测试文件 → A/B/C三级分类（行业深度/专题研究/商品分析），非二元过滤，人工终审
⚙️ 阶段3 · 清洗入库：OCR文本提取 → LLM四维提取（实体/指标/框架/逻辑链）→ 多模型降级链（MiniMax→sensenova→Qwen）→ PG+Qdrant双写
🛡️ 阶段4 · 12项质量门禁：thinking泄漏=0 / 模板化表达<5% / 三元组断裂=0 / Wiki同步率>95% / 描述句号缺失=0 等自动校验
关键迭代：06.03 API Key检测误判（Python子进程不加载.bashrc）→ 制定强制前置检查清单。07.05 本地研报筛选方法升级（手工13份→系统扫描108份）。OCR提取率 V0→V3 四轮迭代 30%→90%+。
可复用模板：研报客API搜索 → 五维加权初筛 → 本地二级筛选+三级分类 → OCR清洗去重 → 四维提取入库 → 12项质量门禁 → Wiki自动更新。

📌 核心认知：这 4 套工作流不是"星恒专用"——换一个项目名，流程完全一样。你今天带走的就是这 4 个模板。
